# Contributing to CourseTracker

Thank you for your interest in contributing to **CourseTracker**! 🎉  
We welcome contributions from everyone—bug reports, feature requests, documentation improvements, and code contributions are all highly appreciated.  

---

## 1️⃣ How to Contribute

### Step 1: Fork the repository
Click the **Fork** button at the top right of the repository page to create your own copy of the project.

### Step 2: Clone your fork
```bash
git clone https://github.com/your-username/CourseTracker.git
cd CourseTracker
```
### Step 3: Create a branch
 
Create a new branch for your work:
```bash
git checkout -b feature/awesome-feature
```
### Step 4: Make changes

- For code: follow the existing structure (cli.py, gui.py, parsers/, database.py)

- For documentation: edit README.md or add new docs in a clear, structured format

- For bug fixes: include steps to reproduce and your fix

### Step 5: Commit changes

Use clear commit messages:
```bash
git add .
git commit -m "Add feature: search multiple platforms at once"
```
### Step 6: Push your branch 
```bash
git push origin feature/awesome-feature
```
### Step 7: Open a Pull Request

- Go to your fork on GitHub

- Click Compare & Pull Request

- Write a clear description of your changes

- Submit the PR

### 2️⃣ Guidelines

- Follow PEP8 style for Python code

- Keep functions and methods modular and readable

- Update documentation if your changes introduce new functionality

- Test new features before submitting

### 3️⃣ Reporting Bugs

If you find a bug:

1.Check existing issues to see if it’s already reported

2.Create a new issue with
- Title describing the problem

- Steps to reproduce

- Expected vs actual behavior

- Screenshots if possible

### 4️⃣ Suggesting Features

- We love feature suggestions! Please provide:

- A clear description of the new feature

- The problem it solves or the improvement it brings

- Optional mockups or examples

### 5️⃣ Code of Conduct

-By contributing to this project, you agree to follow our Code of Conduct:

- Be respectful to all contributors

- Communicate clearly and politely

- Be open to feedback and collaboration

**This file helps maintain a healthy and organized workflow for CourseTracker, and ensures your contributions are valuable and easily merged.**
